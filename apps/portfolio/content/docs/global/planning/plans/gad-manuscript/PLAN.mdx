# gad-manuscript — fiction and book development sub-CLI

## Summary

`gad-manuscript` is a domain `.gad` pack and sub-CLI for fiction and book development. It extends GAD with book-specific planning primitives: beat sheets, scene tracking, canon locks, forward questions, and chapter execution loops.

**Source:** `MagicbornStudios/gad-manuscript` → `vendor/gad-manuscript`

**Status:** brand new repo — not started. Work begins after GAD is stable and no longer depends on GSD.

---

## Dependency on GAD stability

gad-manuscript work begins only when:
1. GAD is fixed up and GSD is no longer the active planning system
2. Reusable CLI code is migrated to mb-cli-framework
3. `gad` CLI is finished
4. Portfolio, grime-time, and repo-planner are fully using mb-cli-framework

Until those gates are cleared, `rp-manuscript` stays in its current form and is not migrated.

## Sub-CLI registration

When gad-manuscript is installed, it registers itself with the `gad` CLI:

```
gad manuscript beats [--projectid] [--json|--md]
gad manuscript beats --create "Act I — Call to Refuse" [--projectid]
gad manuscript beats --update <beatid> [--status]

gad manuscript scenes [--phaseid] [--beatid] [--projectid]
gad manuscript scenes --create "The Tournament" [--phaseid] [--beatid]

gad manuscript canon-locks [--projectid]
gad manuscript canon-locks --create "Mordred's motivation is revenge" [--locked-by <phaseid>]

gad manuscript forward-questions [--projectid]
gad manuscript forward-questions --create "Does Guinevere know?" [--phaseid]
gad manuscript forward-questions --update <qid> --status resolved

gad manuscript extract --from STATE.md [--projectid]   # parse book state into structured output
gad manuscript chapter-status [--projectid]            # summary of chapter completion
```

The `gad.json` manifest declares the sub-CLI:
```json
{
  "name": "manuscript",
  "version": "0.1.0",
  "extends": "get-anything-done",
  "min_gad_version": "1.31.0",
  "subcli": {
    "name": "manuscript",
    "entry": "cli/manuscript-main.ts",
    "description": "Book development commands — beats, scenes, canon locks"
  },
  "domain": "fiction-creative-writing",
  "token_estimate": {
    "install_tokens": 4800,
    "per_phase_tokens": 2100
  }
}
```

## Skills to migrate from rp-manuscript

`rp-manuscript` (currently in `vendor/repo-planner/skills/rp-manuscript/SKILL.md`) covers:

| rp-manuscript concept | gad-manuscript equivalent |
| --- | --- |
| Story vocabulary table | `gad.json` state extension schema |
| Section vs per-novel planning layers | GAD config + sub_repos monorepo awareness |
| Fiction state file structure | `STATE.md` extensions template |
| Fiction task IDs | task ID conventions in `gad.json` |
| Canon locks pattern | `gad manuscript canon-locks` command |
| Forward questions lifecycle | `gad manuscript forward-questions` command |
| Chapter execution loop | GAD `execute-phase` + chapter done checklist workflow |
| 10-item chapter done checklist | workflow template |
| Verification gates | GAD `verify-work` + manuscript extensions |
| EPUB build gate | new manuscript-specific gate |
| Per-book AGENTS.md | gad-manuscript `AGENTS.md` template |

Each of these becomes a skill, workflow, template, or command in the gad-manuscript pack structure.

## Eval project: mordreds-tale-bare

**Location:** `vendor/gad-manuscript/evals/mordreds-tale-bare/`

**Source requirements:** Mordred's Tale book requirements (from the portfolio/books section).

**What it tests:**
- Does the agent correctly use beat sheets when planning?
- Are canon locks maintained across phases?
- Are forward questions resolved before chapters close?
- Is scene structure correct?
- Is the chapter done checklist followed?

**Scoring dimensions** (all standard GAD dimensions plus):

| Domain-specific metric | Definition |
| --- | --- |
| Beat coverage | Fraction of required beats addressed in implementation |
| Canon consistency | No canon violations across the full run |
| Scene structure adherence | Scenes follow the expected arc structure |
| Forward question resolution rate | Forward questions resolved before chapter closes |
| Chapter checklist completion | All 10 checklist items checked per chapter |
