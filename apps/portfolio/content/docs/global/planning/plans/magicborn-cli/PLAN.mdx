# Magicborn CLI ‚Äî is it a good idea?

**Yes**, for *this* repo: you already run **many** `pnpm` scripts (auth seed, RAG ingest, book build, media publish, provider bootstrap). A **single entry** with **consistent flags**, **`--help`**, **config discovery**, and **tabular output** reduces cognitive load and avoids duplicating Payload/env bootstrapping in one-off scripts.

## Recommended shape

| Principle | Why |
| --- | --- |
| **One binary, subcommands** | `magicborn users list`, `magicborn media prompts`, `magicborn books status` ‚Äî matches mental model of `kubectl`, `flyctl`, `pnpm`. |
| **Thin wrappers first** | Delegate to existing `tsx` scripts and `lib/` until a command needs bespoke behavior. |
| **Shared runtime** | Reuse `loadScriptEnv`, `getPayloadClient`, logging ‚Äî same as `invite-user` / `rag-ingest-runner`. |
| **JSON + table modes** | `--json` for automation; default human tables for terminal UX. |

## Candidate command groups (backlog)

| Group | Examples |
| --- | --- |
| **auth** | `users list`, `users get --email=`, `invite` (wrap `auth:invite`) |
| **books** | `books list`, `books missing-covers`, `books build`, `books publish` |
| **media** | `media prompt show`, `media generate --scene=‚Ä¶`, `media gaps` (records without `imagePrompt` metadata ‚Äî needs schema) |
| **data** | `rag ingest`, `rag status`, `provider export` |
| **planning** | `planning packs list`, `planning embed` |

## Architecture notes

- Implement with **`commander`** or **`yargs`** + **`tsx`** shebang, or a small **`packages/magicborn-cli`** workspace package consumed as `pnpm mb`.
- **Max UX**: top-level `magicborn --help`, **per-command help**, **exit codes**, **NO_COLOR** / **FORCE_COLOR**, and **stable stderr** for errors.

### Landed (v0)

- **Root:** `pnpm magicborn` / `pnpm mb` ‚Üí `packages/magicborn-cli` (Commander), **`prepare`** builds `dist/` for the bin.
- **`generate`:** nested commands `book` \| `app` \| `planning-pack` \| `listen` ‚Üí `apps/portfolio/scripts/magicborn/run.ts`. Shared **`lib/magicborn/openai-image-generate`** with **`POST /api/media/generate`**. **`.env`** via `load-magicborn-env` (no `@next/env` in tsx). Output: **`.magicborn/runs/<id>/`** (`image.png`, `prompt.txt`, `manifest.json`). Flags: **`--dry-run`**, **`--print-prompt`**, **`--raw`**, **`--size`**, **`--slot`**, **`--json`**. Terminal UX mirrors **`rag:ingest`** when **`MAGICBORN_CLI=1`** (set by the package forwarder).
- **`vendor add`:** GitHub URL ‚Üí dry-run instructions or **`--apply`**: `git submodule add` + **`pnpm-workspace.yaml`** entry (see **`vendor/repub-builder`**); npm/GitHub Packages notes point at root **`package:*`** scripts ‚Äî no secrets in repo.

### Media asset taxonomy (operator + site)

Every generated file maps to a **stable ref** for CLI, local working files, and Payload/storage: **`kind` + record id (+ optional sub-id / variant)**. Kinds align with what the app already surfaces or will surface soon.

| Kind | Typical use | Text in image? | Notes |
| --- | --- | --- | --- |
| **`site`** | Logos, favicon-related art, shared chrome | **Logo:** usually no embedded tagline unless intentional; wordmarks are a separate deliberate choice | **Active logo** (and similar) is an **explicit pointer** set by the operator ‚Äî not ‚Äúlatest upload wins‚Äù (**`global-tooling-dec-03`**). |
| **`book`** | Covers, scene stills, character sheets | **Covers:** yes ‚Äî title + subtitle (and series if needed) per brief | Scene/character images often **no** burned-in text; captions live in CMS/copy. |
| **`app`** | Catalog cards, hero imagery for `/apps` | Often **yes** ‚Äî app name + short subtitle for card-style art | Matches ‚Äúbook cover with text‚Äù energy for discoverability. |
| **`project`** | Project covers / thumbnails | Often **yes** ‚Äî project title + optional tagline | Same pattern as apps where the grid is image-forward. |
| **`song`** / **`listen`** | Listen row art, BandLab-linked visuals | Case-by-case; default **no** text unless promoting a track title on the art | Keep flexible per row. |
| **`planning-pack`** | Pack thumbnails, modal art | Usually **no** or minimal; title in UI | Align with planning embed surfaces. |
| **`blog`** / **`article`** | Post heroes, OG-style images | Optional title on art for share cards; editorial choice | Not all posts need text on image. |
| **`ui`** | Backgrounds, textures, marketing shells | Typically **no** text | Style layer + composition only. |

**Prompt architecture (two layers):** (1) **Magicborn media design style** ‚Äî shared art direction (repo `lib/magicborn-prompts` first; optional Payload override later). (2) **Asset prompt** ‚Äî the specific scene, product, or layout brief. Image models that must **avoid** text still use the global negatives from the style block; when text **is** wanted, the asset prompt states typography intent explicitly.
