# RP migration â€” GAD MD schema, GUI-only RepoPlanner

## Goal

Retire the RP XML planning schema and `rp-*` skills in favor of GAD's Markdown-based schema and `gad:` skill prefix. RepoPlanner becomes the official GUI frontend for GAD state, not a planning methodology on its own.

## Context

`global-planning-01` through `global-planning-03` established:
- GAD CLI binary (`gad`) with workspace/projects/state/phases/tasks/docs/eval commands
- GAD eval framework (TRACE + SCORE + drift metrics)
- `lib/state-reader.cjs` and `lib/roadmap-reader.cjs` with XML + MD fallback

The `.planning/` directory still uses XML. RepoPlanner reads XML. The `rp-*` skills duplicate `gad:` skills.

## Work items

### 1. XML -> Markdown migration (blocking: all in-progress tasks must be done first)

**Prerequisite:** `TASK-REGISTRY.xml` tasks `02-01`, `03-02`, `05-01` are still `in-progress`.
Do not run `gad migrate-schema --yes` until these are done or explicitly closed.

When ready:
```bash
node vendor/get-anything-done/bin/gad.cjs migrate-schema --yes
```

Then manually merge `TASK-REGISTRY.xml` data into `STATE.md` (the CLI archives but does not migrate task registry).

### 2. Remove redundant rp-* skills

Per the gad-architecture plan, these `rp-*` skills are superseded by `gad:` equivalents:

| Remove | GAD equivalent |
| --- | --- |
| `rp-add-todo` | `gad:add-todo` |
| `rp-check-todos` | `gad:check-todos` |
| `rp-debug` | `gad:debug` |
| `rp-execute-phase` | `gad:execute-phase` |
| `rp-map-codebase` | `gad:map-codebase` |
| `rp-milestone` | `gad:audit-milestone` + `gad:new-milestone` + `gad:complete-milestone` |
| `rp-new-project` | `gad:new-project` |
| `rp-plan-phase` | `gad:plan-phase` |
| `rp-quick` | `gad:quick` |
| `rp-session` | `gad:pause-work` + `gad:resume-work` + `gad:thread` |
| `rp-verify-work` | `gad:verify-work` |

The `rp-*` skills live in `vendor/repo-planner`. They are not deleted from the repo but should be marked deprecated with a pointer to the `gad:` equivalent. The skill system will still find them, but the user should prefer `gad:` prefix.

### 3. Update AGENTS.md files

- **`.planning/AGENTS.md`** -- done in global-planning-04: GAD CLI table added, migration gate documented.
- **Root `AGENTS.md`** -- add note that `gad:` is now the preferred skill prefix, `rp-*` is legacy.
- **Section `AGENTS.md` files** -- align on `gad:` where they reference `rp-*`.

### 4. Update vendor/repo-planner to read GAD MD state

After the XML migration, RP's planning file discovery needs to handle `.md` files in addition to `.xml`. This is a vendor submodule change in `vendor/repo-planner`.

### 5. RepoPlanner skill deprecation notices

In each `rp-*` skill file, prepend:
```
> **Deprecated:** Use `gad:<equivalent>` instead. This skill remains for backwards compatibility.
```
