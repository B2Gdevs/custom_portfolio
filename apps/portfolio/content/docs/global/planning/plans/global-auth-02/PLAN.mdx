# Kickoff: `global-auth-02`

| --- | --- |
| `goal` | Refresh the site shell around the existing owner auth/session contract so the portfolio feels like a real SaaS account surface without opening public signup or multi-user management yet. |
| `scope` | sidebar footer account card, live session fetch, avatar in phase one, user email/display name, role, org name, tenant slug/id, logout action, and owner-only private/admin entry points driven by entitlements instead of `NODE_ENV`. |
