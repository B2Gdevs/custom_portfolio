## Unpushed range blocker

The current `origin/main..main` range is still **six commits ahead**, and the largest blobs in that unpushed range are already enough to block or slow a normal `git push`.

| Blob | Approx size | Current path | Consequence |
| --- | --- | --- | --- |
| `portfolio.db` | `113 MB` | `apps/portfolio/portfolio.db` | this local DB is still present in the unpushed commit range, so deleting it later in the branch does not make the push safe by itself |
| `mordreds_tale` EPUB variants | `~41 MB` each | `apps/portfolio/public/books/mordreds_tale*/book.epub` | generated book binaries are still riding inside the pending history |
| lockfile snapshots | `600-700 KB` | `pnpm-lock.yaml` | not a blocker alone, but shows the push range is carrying large churn even outside binary artifacts |

That means `global-data-02-02` has to cover **history cleanup or branch rebuild from `origin/main`** as part of the storage migration, not only future artifact policy.
