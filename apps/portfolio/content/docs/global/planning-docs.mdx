# Planning Docs

## Registry

| Record | Purpose |
| --- | --- |
| `state` | Current cycle, focus, and constraints for the **Global** section |
| `task-registry` | Tasks that change global docs IA, cross-links, or repo-wide planning UX |
| `errors-and-attempts` | Approaches we tried and abandoned for this section |
| `decisions` | Stable decisions about Global vs section planning layout |
| **Reference (not in this folder)** | **[Global planning guide](/docs/global/global-planning)** — layers, task id shape, **Mermaid** workflows, planning-pack policy, RepoPlanner boundaries |

## Loop

| Step | Rule |
| --- | --- |
| `1` | Update **state** / **task-registry** before changing global IA or `docs.ts` grouping |
| `2` | Execute one logical task (link sweep, new global reference page, nav behavior) |
| `3` | Verify docs build and the file tree shows **planning** vs **reference** correctly |
| `4` | Record outcomes in **state** and mark **task-registry** rows |

## Agent context (read order)

For **Global-section** work (this folder), read this page first, then [state](/docs/global/state), [task registry](/docs/global/task-registry), [decisions](/docs/global/decisions), [errors and attempts](/docs/global/errors-and-attempts) when relevant.

For **diagrams, id grammar, Mermaid, and planning-pack policy**, read the **[Global planning guide](/docs/global/global-planning)** — it is a **reference** page in the same section, not inside the **planning** subtree.

For **monorepo-wide** gates, read root **[AGENTS.md](../../../../../AGENTS.md)**, **[documentation/requirements](/docs/documentation/requirements)**, **[documentation/state](/docs/documentation/state)**, **`.planning/ROADMAP.xml`**, **`.planning/TASK-REGISTRY.xml`**, **`.planning/STATE.xml`**. Repo-root **`REQUIREMENTS.md`** is a stub pointer only.
