# Planning Docs

## Registry

| Record | Purpose |
| --- | --- |
| `global-planning` | repo-wide vs section planning model, id namespaces, cross-references, Mermaid workflow diagrams |
| `state` | current cycle, focus, and constraints for the documentation section |
| `task-registry` | active and queued documentation-site work |
| `errors-and-attempts` | notable failed approaches and restructures |
| `decisions` | stable decisions for docs IA, requirements, and planning behavior |

## Loop

| Step | Rule |
| --- | --- |
| `1` | update these documentation planning docs before site-structure work |
| `2` | execute one logical docs-site or requirements task |
| `3` | verify docs rendering and app build output |
| `4` | write the new state back into these records |

## Agent context (read order)

For **documentation-section** work, read this page first, then [global planning](/docs/documentation/global-planning) (namespaces, references, diagrams), then [state](/docs/documentation/state), [task registry](/docs/documentation/task-registry), [decisions](/docs/documentation/decisions). For **monorepo-wide** gates, read [.planning/REQUIREMENTS.md](../../../../../.planning/REQUIREMENTS.md) and [.planning/IMPLEMENTATION_PLAN.md](../../../../../.planning/IMPLEMENTATION_PLAN.md), then root **`AGENTS.md`** at the repository root.
