# Kickoff: `documentation-site-14`

## Goals

1. **Site sidebar + reader rail** — Eliminate conflicting layout behavior between the portfolio **shadcn `SidebarProvider`** (`SiteLayout` / `Nav`) and the **repub-builder `ReaderWorkspaceSidebar`** (expand/collapse + `READER_NAV_EXPANDED_KEY`). Observed: narrow viewport / DevTools dock → **wide empty rail** without icons; navigating reader → **site sidebar missing** while reader rail expanded; **expand** appears to paint reader content into the **wrong** horizontal slot.
2. **Root `global-error`** — `app/global-error.tsx` is the **Next.js root error boundary**; it replaces the shell on **uncaught render errors** in the tree under `layout.tsx`. Production shows the generic line until **`NEXT_PUBLIC_CLIENT_ERROR_DEBUG=1`** or dev — capture **digest** from Vercel logs and fix the **underlying** throw (often `useSidebar` outside provider, hydration, or a client component exception during shell render).
3. **Mordred’s Tale EPUB 404** — `public/books/manifest.json` points at **`/api/published-book-artifacts/file/<filename>.epub`**. The route (`app/api/published-book-artifacts/file/[filename]/route.ts`) resolves **Payload `publishedBookArtifacts`** by filename, then **local disk** under `media/published-book-artifacts/`, then **S3** key candidates, then **redirect** to `public/books/<slug>/book.epub` if present. A 404 with the file “visible in storage” usually means **DB row filename ≠ manifest string**, **S3 key mismatch**, or **build/publish** never ran on that environment — **not** the Postgres adapter itself (Payload already uses **`@payloadcms/db-postgres`** / Supabase-compatible Postgres).
