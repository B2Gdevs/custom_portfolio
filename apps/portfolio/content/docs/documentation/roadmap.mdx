# Roadmap

## Sections

| Section | Status | Focus | Next |
| --- | --- | --- | --- |
| `books` | `active` | stabilize the reader shell and keep the book pipeline centered on EPUB3 output | improve page formatting and polish the dedicated reader experience |
| `magicborn` | `active` | establish a worldbuilding section for the world of Magicborn using short story style lore docs | grow the lore corpus, keep canon boundaries clear, and feed public chat from these docs instead of raw manuscript pages |
| `dialogue-forge` | `active` | keep the editor and exporter docs current with the shipped toolchain | continue implementation-specific docs and cleanup as the tool evolves |
| `blog` | `active` | keep the blog as a public destination and frame RichEPub as an archived experiment with tooling lessons | align blog categories and future editor updates with the docs tree |
| `projects` | `active` | create a first-class planning surface for the projects showcase and align it with the coming blog discovery/article UX pass | define searchable/filterable project browsing, detail-page structure, and repo/app/download metadata |
| `listen` | `active` | treat **`/listen`** as a first-class discovery surface for tracks and BandLab presets | implement catalog model, blog/projects-style filters, and group-password locks per [listen task registry](/docs/listen/task-registry) |
| `documentation` | `active` | make the docs site itself easier to understand and maintain | improve section discoverability and keep roadmap summaries current |
| `editor` | `planned` | define a writing/editor tool that can compile authored content to cleaner EPUB3 | validate scope, authoring workflow, and source-format requirements |
