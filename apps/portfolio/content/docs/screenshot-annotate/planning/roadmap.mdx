# Roadmap

**Constraint (all phases):** user screenshots and annotations stay **client-only** — no DB, no Payload media for user content ([requirements](/docs/screenshot-annotate/requirements#privacy-and-persistence-non-negotiable)).

**Stack:** [tldraw SDK](https://tldraw.dev/) — [Image annotator pattern](https://tldraw.dev/examples/image-annotator), [Clipboard](https://tldraw.dev/sdk-features/clipboard).

| Phase | Goal |
| --- | --- |
| **annotate-01** | Route + **minimal chrome**; **`pick` → `annotate` → `export`** per tldraw example; **pick**-phase clipboard + **annotate**-phase tldraw clipboard APIs; **four tools** (Select, Hand, Rectangle/geo, Text); **PNG** export only; **Payload** `site-app-records` **catalog row only**. |
| **annotate-02** | Product polish on top of tldraw: stroke width, color pickers, **solid** “hide” preset; z-order / context actions via SDK where possible. |
| **annotate-03** | **Camera / viewport** — **done** for image lock (`TLCameraOptions.constraints`); optional: further tldraw chrome trimming; **Apps hub** skeleton when Payload is slow (still open if needed). |
| **annotate-04** | **Tesseract.js** OCR spotlight + copy (shipped); optional follow-ups: word-level boxes, extra languages, **`04-04`** redact presets. |
