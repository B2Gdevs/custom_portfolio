# RichEPub (.repub) Format Specification

RichEPub is a container format for ebooks that support React, Tailwind CSS, and npm dependencies. The file extension is **.repub**. It is not compatible with standard EPUB readers; it is intended for web-based readers that can unpack and run the bundled content in a sandboxed environment.

## Container Format

A **.repub** file is a **ZIP archive** with a defined internal layout. All paths use forward slashes.

### Directory Layout

```
.repub (ZIP root)
├── repub.json          # Required: manifest (format version, metadata, entry, dependencies)
├── content/            # Entry HTML and optional static assets
│   └── index.html      # Required: entry document; loads assets from ../assets/
├── assets/             # Pre-built JS and CSS (no node_modules)
│   ├── index-*.js      # Bundled JavaScript
│   └── index-*.css     # Compiled CSS (e.g. Tailwind)
└── meta/               # Optional: metadata for catalog/UI
    ├── cover.png       # Optional: cover image
    └── description.txt # Optional: short description
```

- **repub.json** (root): Single required manifest file. Describes format version, book metadata, entry path, declared dependencies, and build info.
- **content/**: The entry HTML and any other static files that the entry references (e.g. images). The entry document (typically `content/index.html`) must load scripts and styles from **assets/** using relative paths (e.g. `../assets/index-abc123.js`).
- **assets/**: Output of the build step. Only pre-built JS bundle(s) and CSS. No runtime npm resolution; everything is bundled at build time.
- **meta/** (optional): Cover image, description, etc., for use by the portfolio or other readers when listing the book.

## repub.json Schema

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `formatVersion` | number | Yes | Format version (e.g. `1`). Readers may reject unknown versions. |
| `title` | string | Yes | Book title. |
| `author` | string | No | Author name. |
| `entry` | string | Yes | Path to the entry HTML file, relative to ZIP root (e.g. `content/index.html`). |
| `dependencies` | object | No | Map of npm package name to version string (e.g. `{ "react": "^18.2.0", "tailwindcss": "^3.4.0" }`). Declarative only; used for documentation and audit. |
| `buildInfo` | object | No | Builder metadata. |
| `buildInfo.builderVersion` | string | No | Version of the tool that produced this .repub. |
| `buildInfo.buildTime` | string | No | ISO 8601 build timestamp. |

### Example repub.json

```json
{
  "formatVersion": 1,
  "title": "My Book",
  "author": "Author Name",
  "entry": "content/index.html",
  "dependencies": {
    "react": "^18.2.0",
    "react-dom": "^18.2.0",
    "tailwindcss": "^3.4.0"
  },
  "buildInfo": {
    "builderVersion": "1.0.0",
    "buildTime": "2025-03-11T12:00:00.000Z"
  }
}
```

## Dependency Field Semantics

- **Declarative**: The `dependencies` object describes what the author used when building the book. It is typically derived from the repub project’s `package.json` (and optionally lockfile) at pack time.
- **No runtime resolution**: The .repub file does not contain `node_modules`. All dependencies are resolved and bundled during the build step. The reader never fetches npm.
- **Audit/documentation**: Readers or tooling can display or validate the list of packages for security or compatibility.

## Versioning

- **formatVersion**: Increment when the container layout or `repub.json` schema changes in a backward-incompatible way. Readers should reject archives with an unsupported `formatVersion`.
- **builderVersion**: Optional; identifies the builder that produced the file for debugging and compatibility.

## Relation to EPUB

- **EPUB**: Standard format (HTML/CSS, no React, no npm). Use for universal reading (Kindle, Apple Books, etc.).
- **RichEPub (.repub)**: Extended format for web-only readers that support React, Tailwind, and npm. Not intended to be compatible with standard ebook readers.
