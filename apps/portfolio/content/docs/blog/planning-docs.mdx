# Planning Docs

## Registry

| Record | Purpose |
| --- | --- |
| `state` | current cycle, focus, constraints, and active phase ids for the blog section |
| `task-registry` | active and queued blog work in execution order |
| `errors-and-attempts` | notable failed approaches and course corrections |
| `decisions` | stable section decisions for blog structure and publishing |

## Loop

| Step | Rule |
| --- | --- |
| `1` | update these blog planning docs before implementation |
| `2` | execute one logical blog or publishing task |
| `3` | verify the site build and relevant content rendering |
| `4` | update the blog records again before moving on |

## Id Rule

| Field | Value |
| --- | --- |
| `phase-id` | use `blog-<stream>-<phase>` for blog planning phases |
| `task-id` | use `blog-<stream>-<phase>-<task>` for executable work |
| `attempt-id` | use `blog-<stream>-<phase>-attempt-<task>` for notable course corrections |

## Current Streams

| Stream | Purpose |
| --- | --- |
| `content` | publishing strategy, categorization, and editorial direction |
| `discovery` | index-page search, filters, sort, command palette, and article chrome |
| `identity` | shared content-shell reuse with section-specific theme treatment, mini logo, and orientation cues |
