# Repo Planner — Decisions

## `REPOPLAN-SHELL-01` — Cockpit shell lives in the imported package

| Field | Value |
| --- | --- |
| **Decision** | **Project / pack switcher**, **upload pack**, **export workspace**, **init-template downloads**, and the **dashboard chrome** around **`PlanningCockpit`** are **owned by the `repo-planner` package** (vendored submodule) and **imported** into the portfolio. The host app **does not** keep a long-term parallel shell such as **`PlanningCockpitDashboard`** once the package exports a single **host-mountable shell** (name TBD upstream: e.g. full layout wrapping `organisms/planning-cockpit`). |
| **Rationale** | One UX surface, one place to fix bugs; portfolio only supplies **routes**, **API handlers**, **env**, and **modal/book props**. |
| **Consequences** | **`repo-planner-integration-03`** includes **upstream (or transitional) extraction** of that shell; portfolio page becomes a thin **`import { … } from 'repo-planner'`** + layout wrapper. Interim: portfolio shell remains only until the package ships the export. |

## `REPOPLAN-EMBED-01` — Public embed is read-only on the server

| Field | Value |
| --- | --- |
| **Decision** | **Embedded cockpit** ([`/apps/repo-planner`](/apps/repo-planner)): **GET** routes may read committed **`.planning/`** (today via CLI JSON; planned XML/read path). **`POST /api/planning-edits/apply`** and **`POST /api/planning-cli/run`** return **403** unless **`PLANNING_COCKPIT_ALLOW_SERVER_WRITES=1`**. No portfolio-hosted planning LLM; **Chat** tab stays inert unless the operator wires **`POST /api/ai/planning-chat`**. |
| **Rationale** | Same spirit as the EPUB reader: **browser-local / export → git**, not casual server mutation. |
| **Consequences** | Operators use **`pnpm planning`** / **standalone RepoPlanner** for write-capable workflows. |

| Mechanism | Portfolio default |
| --- | --- |
| **`POST /api/planning-edits/apply`** | **403** — enable only with **`PLANNING_COCKPIT_ALLOW_SERVER_WRITES=1`** |
| **`POST /api/planning-cli/run`** | **403** — same flag |
| **`GET /api/planning-state`** | **On** — reads bundle (CLI-backed today) |

## `REPOPLAN-REPORTS-01` — CLI telemetry outside `.planning/`

| Field | Value |
| --- | --- |
| **Decision** | **`REPOPLANNER_REPORTS_DIR`** defaults to **`.planning-reports/`** at repo root (via **`scripts/run-planning-cli.mjs`** and **`next.config.ts`** / **`getRepoPlannerChildEnv()`**) so **`usage.jsonl`** and generated reports do not recreate **`.planning/reports/`** after minimal init. |
| **Rationale** | Bare **`.planning/`** stays XML + config only; reports are local operator artifacts. |
