# Kickoff Example

## Kickoff: `repo-planner-integration-05`

| Field | Notes |
| --- | --- |
| `goal` | Make the planning loop first-class in RepoPlanner so agents can pick, start, and close phases with consistent workflow guidance. |
| `scope` | Shared recommendation scoring, kickoff output, done-gate checks, ownership guidance, sprint context, and cockpit workflow panels across CLI, Live, and Pack. |
| `non-goals` | Full structured editing, autosave, immutable-id migration flows, or raw markdown replacement outside the workflow surface. |
| `dependencies` | Shared live/pack bundle contract from phase `03`, roadmap-first section docs, and the repo-wide loop in [AGENTS.md](/AGENTS.md). |
| `tests required` | Workflow ranking fixtures, CLI workflow command coverage, cockpit rendering tests, and done-gate refusal coverage for executable work without tests. |
| `definition of done` | RepoPlanner shows ranked recommendations, emits kickoff guidance, blocks invalid done transitions for executable work, and renders ownership / discussion / sprint signals in the cockpit with matching CLI data. |
| `first tasks` | Ship shared workflow helpers; expose kickoff, done gate, and ownership details in the CLI; render the cockpit workflow cards; update section planning docs when the phase closes. |
| `open questions` | None remain for this shipped example. |
