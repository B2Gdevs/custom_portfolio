## Phase `repo-planner-integration-03` (vendor cockpit body + data plane + shell upstream)

**Problem today:** [`/apps/repo-planner`](/apps/repo-planner) uses a **placeholder** live pane; workspace chrome is still **host-owned** (`PlanningCockpitDashboard` / `RepoPlannerWorkspaceShell`) while **`REPOPLAN-SHELL-01`** says **upload / pack switcher / export / init downloads** belong in the **`repo-planner`** package long term.

**Desired:**

- **Vendor `PlanningCockpit`** depth (metrics, %, charts, tasks, phases, state) composed in the live pane -- source: [`planning-cockpit.tsx`](https://github.com/MagicbornStudios/RepoPlanner/blob/main/components/organisms/planning-cockpit.tsx).
- **Host-mountable workspace shell** exported from **`repo-planner`** so the portfolio page is a thin **import + layout** (interim host shell until then).
- **Display** can use **parsed XML / on-disk reports** where possible so the UI is not defined by CLI spawn per poll for **read**; **writes** stay gated ([decisions](/docs/repo-planner/decisions) **`REPOPLAN-EMBED-01`**).
- **Pack mode** feeds the **same bundle shape** as live repo (parse `TASK-REGISTRY.xml`, `STATE.xml`, `ROADMAP.xml`, report `.md` in pack).
- **Init template downloads** + **planning pack modal** copy per **`REPOPLAN-PACK-MODAL-01`**.

### Data contract (conceptual)

Vendor UI expects a **bundle** (see `Bundle` in `planning-cockpit.tsx`): `snapshot`, `openTasks`, `openQuestions`, `context.summary.phases`, `agentsWithTasks`, etc. Host implements something like **`CockpitDataSource`**: `getBundle()`, `getMetrics()`, `getLatestReport()`, polling policy.

| Source | `getBundle()` | `getMetrics()` | Report markdown |
| --- | --- | --- | --- |
| **Live monorepo** | `/api/planning-state` (today: CLI JSON) → later XML reader route | `/api/planning-metrics` | `/api/planning-reports/latest` |
| **Uploaded pack** | Client or **`POST /api/planning-pack/parse`** → same JSON shape | Synthetic series or honest empty | First `*.md` in pack or summary from XML |

**Open:** pack metrics placeholder vs counts-from-XML only → task **`03-04`**.
