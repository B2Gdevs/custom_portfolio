# Repo Planner — Getting started

You do not have to use this monorepo's layout to benefit from a **planning loop**: keep a visible **state**, an ordered **task registry**, and a **roadmap** (or phase list). Agents and humans read the same files; ids stay stable in chat and commits.

## Reference implementation

[MagicbornStudios/RepoPlanner](https://github.com/MagicbornStudios/RepoPlanner) provides:

- **XML-shaped** templates under `.planning/templates/` (bundled inside the submodule when you run **full** init).
- **`loop-cli.mjs`** — snapshot, task updates, reports, `setup checklist`, `init` for greenfield bootstrap.
- **Cockpit UI** — embeddable in Next (see upstream **INSTALL.md**). **Policy** for this site: [decisions](/docs/repo-planner/planning/decisions).

## Submodule

| Path | Upstream |
| --- | --- |
| `vendor/repo-planner` | [MagicbornStudios/RepoPlanner](https://github.com/MagicbornStudios/RepoPlanner) |

Update intentionally: `cd vendor/repo-planner && git fetch && git checkout <ref>` then commit the submodule pointer from the host repo.

```bash
git submodule update --init --recursive vendor/repo-planner
```

## Root scripts (repository root)

Defined in root **`package.json`**:

| Script | Purpose |
| --- | --- |
| `pnpm planning` | **`scripts/run-planning-cli.mjs`** → `loop-cli.mjs` (sets **`REPOPLANNER_REPORTS_DIR=.planning-reports`** unless overridden) |
| `pnpm planning:init` | `init --minimal --no-agents-md` — **`.planning/`** = XML + **`planning-config.toml`** + **`.planning/AGENTS.md`**; may create repo-root **`REQUIREMENTS.md`** stub if missing; **canonical** narrative = **[documentation/requirements](/docs/documentation/requirements)** + **[documentation/state](/docs/documentation/planning/state)**; **no** `IMPLEMENTATION_PLAN.md`; **no** `.planning/templates`, `phases/`, `reports/`, narrative `.md` under `.planning/` |
| `pnpm planning:init:force` | Same with `--force` (overwrites bootstrap files in `.planning/`) |
| `pnpm planning:init:full` | Upstream-style init (`templates/`, `phases/01-greenfield/`, `reports/.gitkeep`, `.planning/README.md`) — use only when you need RepoPlanner templates locally |
| `pnpm planning:init:full:force` | Full init with `--force` |
| `pnpm planning:snapshot` | Snapshot / agent context |
| `pnpm planning:checklist` | `setup checklist` |
| `pnpm planning:ui` / `pnpm planning:report-view` | **Local** report viewer (default port from CLI) |
| `pnpm planning:embed-packs` | **`planning pack embed-build`** with this repo’s **`-o`** / **`--docs-dir`** → **`apps/portfolio/public/planning-embed/builtin-packs.json`** (cockpit built-in packs). App **dev** / **build** also run **`apps/portfolio/scripts/build-planning-embed-packs.cjs`**, which calls the same **`vendor/repo-planner/scripts/lib/embed-builtin-packs-build.mjs`** implementation. |
| `pnpm planning pack embed-build` | **`--help`** for flags. Requires **`-o, --out <file>`**; optional **`--docs-dir`**, **`--planning-dir`**, **`--docs-path-prefix`**. Node API: **`repo-planner/embed-builtin-packs-build`** (`runPlanningEmbedBuildSync`, `buildEmbedBuiltinPacksPayload`). |
| `pnpm planning -- --version` | Prints **`vendor/repo-planner/package.json`** `version` (root wrapper strips a stray **`--`** from **`pnpm`** forwarding). Same as **`pnpm run planning -- --version`**. |

**CLI dependencies** (`commander`, `fast-xml-parser`, `@iarna/toml`, `ejs`, `gray-matter`) are installed at the **workspace root** (and **`gray-matter`** is listed under **`vendor/repo-planner`** for clarity) so `node` can resolve them when executing the submodule script.

### Quick commands

```bash
pnpm planning setup checklist
pnpm planning snapshot
pnpm planning:ui
```

Pass **`--root <path>`** or set **`REPOPLANNER_PROJECT_ROOT`** if the CLI must resolve a different project root than **`process.cwd()`**. Set **`REPOPLANNER_REPORTS_DIR`** (relative to project root or absolute) for **`usage.jsonl`** / report **`latest.md`** — this monorepo defaults to **`.planning-reports/`** via **`scripts/run-planning-cli.mjs`** and **`next.config.ts`**. See [decisions](/docs/repo-planner/planning/decisions) **`REPOPLAN-REPORTS-01`**.

**Brownfield / minimal bootstrap:**

```bash
pnpm planning:init
```

Creates **`.planning/`** for **XML** + **`planning-config.toml`** + **`.planning/AGENTS.md`**. Execution lives in **`ROADMAP.xml`** / **`TASK-REGISTRY.xml`** / **`STATE.xml`** — not `IMPLEMENTATION_PLAN.md`. Stub **`REQUIREMENTS.xml`** points at documentation paths.

## MDX section docs vs XML

- **Public narrative + section registries:** `apps/portfolio/content/docs/<section>/planning/*.mdx` for planning records, with non-planning reference docs beside that folder.
- **Structured agent loop (RepoPlanner):** `.planning/*.xml` + `planning-config.toml` after **`pnpm planning:init`**. **Monorepo narrative + cross-cutting queue:** **[documentation/requirements](/docs/documentation/requirements)** and **[documentation/state](/docs/documentation/planning/state)**.

## Planning files: what to use when

| File | Role |
| --- | --- |
| **`ROADMAP.xml`** | Product / theme timeline — milestones and phases as the cockpit expects. |
| **`TASK-REGISTRY.xml`** | Tasks — goals, keywords, verification commands, status. |
| **`STATE.xml`** | Pointer — `current-phase`, `next-action`, optional agent registry. |
| **[documentation/requirements](/docs/documentation/requirements)** + **[documentation/state](/docs/documentation/planning/state)** | **Monorepo narrative + cross-cutting queue** (canonical). Repo-root **`REQUIREMENTS.md`** is a short pointer only. |
| **`.planning-reports/`** | **Local only** — CLI usage + generated reports when **`REPOPLANNER_REPORTS_DIR`** / root **`pnpm`** scripts are used. |

**Phases folder** exists only after **`pnpm planning:init:full`**. Default **minimal** layout has **no** `.planning/phases/`.

## Bare `.planning` (default) vs full upstream init

**Contract:** **`.planning/`** is the **root planning section** (XML + **`planning-config.toml`** + **`.planning/AGENTS.md`**). **Human** monorepo narrative lives in **documentation** MDX + **`.planning/*.xml`** for machine loop.

**Default `pnpm planning:init`** creates:

| Under **`.planning/`** | At **repo root** | Omitted (use **`pnpm planning:init:full`**) |
| --- | --- | --- |
| `planning-config.toml`, `STATE.xml`, `TASK-REGISTRY.xml`, `ROADMAP.xml`, `DECISIONS.xml`, `ERRORS-AND-ATTEMPTS.xml`, `REQUIREMENTS.xml` (stub → documentation paths), **`AGENTS.md`** | **`REQUIREMENTS.md`** stub only if bootstrap creates it | `.planning/templates/`, `.planning/phases/`, `.planning/README.md` |

**Local-only:** **`.planning-reports/`** (gitignored). **Former archive:** **`.planning-archive/`** was removed after migration; see **[documentation requirements](/docs/documentation/requirements)** (**Planning archive** section).

**What went wrong in an early `--minimal` pass** (fixed in this repo): narrative **`REQUIREMENTS.md` / `IMPLEMENTATION_PLAN.md`** inside **`.planning/`** broke the bare tree; **`usage.jsonl`** recreated **`reports/`**. Mitigations: **`runPlanningInitMinimal`**, **`REPOPLANNER_REPORTS_DIR`**, **`run-planning-cli.mjs`** defaults — detail in [errors and attempts](/docs/repo-planner/planning/errors-and-attempts).

## Standalone cockpit (operators)

1. Clone [MagicbornStudios/RepoPlanner](https://github.com/MagicbornStudios/RepoPlanner) (or use upstream release / desktop binary).
2. Install dependencies for the **web** app when present (path per upstream README).
3. Point at **this** repository: `REPOPLANNER_PROJECT_ROOT=/absolute/path/to/custom_portfolio` then start the dev server from the RepoPlanner repo.
4. Edit in the cockpit, **export**, and commit from git.

## Embedded cockpit on this site

[`/apps/repo-planner`](/apps/repo-planner) mounts the vendored UI with **read-only server** defaults: **GET** routes may read committed **`.planning/`**; **`POST /api/planning-edits/apply`** and **`POST /api/planning-cli/run`** return **403** unless **`PLANNING_COCKPIT_ALLOW_SERVER_WRITES=1`**. No portfolio-hosted planning LLM. **Workspace chrome** (pack upload, project switcher, export) is **interim host code** today; target is an export from **`repo-planner`** — [Decisions](/docs/repo-planner/planning/decisions) **`REPOPLAN-SHELL-01`**, **`REPOPLAN-EMBED-01`**.
