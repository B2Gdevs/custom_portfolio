# Planning Docs

## Registry

| Record | Purpose |
| --- | --- |
| `state` | current cycle, focus, and constraints for the editor section |
| `task-registry` | active and queued editor-planning work |
| `errors-and-attempts` | failed assumptions and scope corrections |
| `decisions` | stable decisions for the editor direction and authoring workflow |

## Loop

| Step | Rule |
| --- | --- |
| `1` | update these editor planning docs before proposing implementation |
| `2` | execute one logical editor-discovery or prototype task |
| `3` | verify the docs build and any affected site surfaces |
| `4` | write the outcome back into these records |

## Id Rule

| Field | Value |
| --- | --- |
| `phase-id` | use `editor-workflow-<phase>` for editor planning phases |
| `task-id` | use `editor-workflow-<phase>-<task>` for executable work |
| `attempt-id` | use `editor-workflow-<phase>-attempt-<task>` for notable course corrections |
