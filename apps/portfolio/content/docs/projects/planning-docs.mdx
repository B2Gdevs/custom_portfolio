# Planning Docs

## Registry

| Record | Purpose |
| --- | --- |
| `state` | current cycle, focus, constraints, and active phase ids for the projects section |
| `task-registry` | active and queued projects work in execution order |
| `errors-and-attempts` | notable failed approaches and course corrections |
| `decisions` | stable section decisions for projects discovery and showcase structure |

## Loop

| Step | Rule |
| --- | --- |
| `1` | update these projects planning docs before implementation |
| `2` | execute one logical projects or showcase task |
| `3` | verify the site build and relevant listing/detail rendering |
| `4` | update the projects records again before moving on |

## Id Rule

| Field | Value |
| --- | --- |
| `phase-id` | use `projects-<stream>-<phase>` for projects planning phases |
| `task-id` | use `projects-<stream>-<phase>-<task>` for executable work |
| `attempt-id` | use `projects-<stream>-<phase>-attempt-<task>` for notable course corrections |
