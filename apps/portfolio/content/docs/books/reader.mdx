# Book reader (static EPUB & RichEPub)

The portfolio serves built books from **static files** at `/books/<slug>/book.epub` and `/books/<slug>/book.repub`. These are produced by the root-level script `pnpm run build:books` from the **repo-root `books/`** directory.

## Embedded reader in docs

The **BookReaderEmbed** component can be used in any doc or article to show the same reader used on the [Books](/books) page, loading a specific book by slug from those static URLs.

Example (Mordred's Tale):

<BookReaderEmbed slug="mordreds_tale" title="Mordred's Tale" />

## How it works

- **EPUB**: Built from `books/<slug>/chapters/` (markdown → HTML) and written to `apps/portfolio/public/books/<slug>/book.epub`. Users can download it for any EPUB reader.
- **RichEPub**: Same content is packaged as a single HTML document plus assets and zipped as `.repub`. The reader unpacks it and renders in a sandboxed iframe.
- **Manifest**: `build:books` writes `public/books/manifest.json`; the Books list and `getBooks()` read from that manifest.

The reader component is standalone: it fetches `/books/<slug>/book.repub`, unpacks it, and displays the content. No server-side rendering of book content is required.

## Requirements and design

**Requirements** and **design/styling** for books and the reader live in the repo under **`.planning/`**:

- **REQUIREMENTS.md** – Books source of truth, tooling, formats, reader behavior, distribution (GitHub-only).
- **DESIGN_STYLING.md** – Book-like typography, component set (Fumadocs-inspired), theme, future MDX reader.

The implementation plan is in **IMPLEMENTATION_PLAN.md**. For contributors and agents: read `.planning/` for the single source of truth on what we're building and how it should look.
