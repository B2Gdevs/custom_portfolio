## EPUB build: planning supplement

The **`@portfolio/repub-builder`** CLI packs `books/<slug>/` into EPUB. Extra planning material can ride along as **appendix XHTML** in the spine (not listed in the generated table of contents):

| Mechanism | Detail |
| --- | --- |
| CLI | `repub epub <book-folder> --planning <absolute-or-relative-dir>` -- repeat `--planning` for multiple roots. |
| `pnpm run build:books` | Reads optional **`epubPlanningDirs`** in each `book.json` (paths **relative to that book folder**), resolves them, and passes `--planning` for each existing directory. |
| Files included | Recursive `.md`, `.mdx`, `.xml`, `.toml`, `.txt` under each root; skips `node_modules`, `.git`, `dist`, `.next`, `__pycache__`. |
| Mordred's Tale | `books/mordreds_tale/book.json` lists `../../apps/portfolio/content/docs/books/planning` so site planning MDX for that stream ships inside the built EPUB. |

Verify with **`pnpm run build:books`** and open the output under `apps/portfolio/public/books/<slug>/book.epub`.

**CLI productization** (help UX, **`npx` / global install**, publishing): [documentation task registry](/docs/documentation/planning/task-registry) phase **`documentation-site-07-02`**.

## Phases, task ids, and frontmatter

Shape matches [global planning](/docs/global/global-planning): **`<books>-<stream>-<phase>-<task>`** (namespace `books` is fixed for this section).

| Stream | Phases (examples) | Scope |
| --- | --- | --- |
| `reader` | `books-reader-01`, `books-reader-02` | In-app EPUB reader, annotations, read-route planning UI |
| `publishing` | `books-publishing-01` | Editions, build visibility, next live EPUB |
| `ai` | `books-ai-00`, `books-ai-01` | **`books-ai-00`** = historical first public chat on CopilotKit. **`books-ai-01`** = Payload + `sqlite-vec` RAG, citations, owner tooling, and the shipped assistant-ui public shell |

- **Phase id:** `books-<stream>-<phase>` (e.g. `books-reader-02`, `books-publishing-01`, `books-ai-01`).
- **Task id:** `books-<stream>-<phase>-<task>` (e.g. `books-ai-01-04`, `books-reader-01-16`, `books-publishing-01-02`).
- **`repoPath`:** Optional YAML frontmatter on any planning `.mdx` file: repo-relative path to this file (this page sets `repoPath: apps/portfolio/content/docs/books/planning/planning-docs.mdx`). Not shown in the article body.
- **`taskPhase`:** Optional frontmatter when a page is owned by a single phase (e.g. a future `books-ai-01-kickoff.mdx`). Omit on shared indexes like [task registry](/docs/books/planning/task-registry).

Read order for **books** work: root **`AGENTS.md`** -> this page -> [Planning docs](/docs/global/planning/planning-docs) / [Global planning guide](/docs/global/global-planning) (optional) -> [state](/docs/books/planning/state) -> [task registry](/docs/books/planning/task-registry) -> linked book-specific pages; monorepo scope -> [documentation/requirements](/docs/documentation/requirements).
