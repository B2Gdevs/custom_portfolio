## Phases, task ids, and frontmatter

Shape matches [global planning](/docs/global/global-planning): **`<books>-<stream>-<phase>-<task>`** (namespace `books` is fixed for this section).

| Stream | Phases (examples) | Scope |
| --- | --- | --- |
| `reader` | `books-reader-01`, `books-reader-02` | In-app EPUB reader, annotations, read-route planning UI |
| `publishing` | `books-publishing-01` | Editions, build visibility, next live EPUB |
| `ai` | `books-ai-00`, `books-ai-01` | **`books-ai-00`** = shipped site chat (CopilotKit). **`books-ai-01`** = planned Payload + `sqlite-vec` RAG, ingest, citations, owner tooling |

- **Phase id:** `books-<stream>-<phase>` (e.g. `books-reader-02`, `books-publishing-01`, `books-ai-01`).
- **Task id:** `books-<stream>-<phase>-<task>` (e.g. `books-ai-01-04`, `books-reader-01-16`, `books-publishing-01-02`).
- **`repoPath`:** Optional YAML frontmatter on any planning `.mdx` file: repo-relative path to this file (this page sets `repoPath: apps/portfolio/content/docs/books/planning-docs.mdx`). Not shown in the article body.
- **`taskPhase`:** Optional frontmatter when a page is owned by a single phase (e.g. a future `books-ai-01-kickoff.mdx`). Omit on shared indexes like [task registry](/docs/books/task-registry).

Read order for **books** work: this page -> [global planning](/docs/global/global-planning) (optional but recommended once) -> [state](/docs/books/state) -> [task registry](/docs/books/task-registry) -> linked book-specific pages; then [.planning/REQUIREMENTS.md](../../../../../.planning/REQUIREMENTS.md) for reader/tooling scope; root **`AGENTS.md`** for the loop.
