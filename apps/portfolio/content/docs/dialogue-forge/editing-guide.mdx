# Editing Guide

This guide matches the **shipped** editor at **`/apps/dialogue-forge`**.

## Open the app

1. Go to **[Dialogue Forge](/apps/dialogue-forge)**.
2. Edit the **title** in the header; a dot appears after you have unsaved-in-session edits relative to the persisted snapshot.
3. Use the header toggles: **Graph** (default), **Yarn**, **Play**.

## Graph mode

- **Pan and zoom** on the canvas; use the editor’s toolbar for add-node actions, layout, and tools provided by `DialogueEditorV2` (mini-map, controls, etc., as built into the package).
- **Select a node** to edit content, choices, conditions, and flags in the side panel.
- **Connect** NPC outputs to player nodes or other targets using the graph’s edge affordances (choice handles where applicable).
- **Guide / flags** — When the graph UI exposes them, use **Guide** for in-app help and **Flag manager** for schema-aware flag editing.

## Yarn mode

Read the same tree as Yarn-flavored text. Use the editor’s export control to download **`.yarn`** when you need engine-ready output (the app wires export for graph and yarn modes; play mode focuses on simulation).

## Play mode

Run through the dialogue as a player would: choices, flags, and branching behave according to the current tree and flag state.

## Undo and redo

With focus **outside** inputs and contenteditable regions, use **Ctrl+Z** / **Ctrl+Shift+Z** (Windows/Linux) or **⌘Z** / **⌘⇧Z** (macOS) for undo/redo of the **persisted dialogue tree** history managed by the portfolio page (up to 50 steps). The graph may also offer its own undo for canvas operations inside the package.

## Persistence

Work is stored in the browser via the portfolio dialogue-forge store (`localStorage`). Clearing site data clears the dialogue; export Yarn files if you need an offline backup.
