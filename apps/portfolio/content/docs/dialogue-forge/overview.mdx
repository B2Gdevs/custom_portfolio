# Overview

## Problem Statement

Branching dialogue is easy to imagine and hard to keep consistent: writers need both a **bird’s-eye view** of the graph and a **player-shaped** way to feel whether beats land.

## Solution

Dialogue Forge gives you **three views** of the same dialogue tree: a **graph** for structure, **Yarn** text for readability, and **Play** for pacing. You edit NPC lines, player choices, and conditions in one place, then export to **Yarn Spinner** for Unreal.

## Shipped experience

The portfolio app at **`/apps/dialogue-forge`** hosts **`DialogueEditorV2`** from `packages/dialogue-forge`: a React Flow–based graph, optional Yarn and Play modes, and a zustand-backed store so your work survives reloads. See [Architecture](/docs/dialogue-forge/architecture) for paths and data flow.

## Core Concepts

### Dialogue nodes

Nodes are typed **`npc`** (character lines), **`player`** (choices), or **`conditional`** (if / elseif / else style branches). They connect via edges; positions are stored so the graph layout stays stable.

### Memory flags

Choices and nodes can set flags; choices can require flags or comparisons so options appear only when the story state matches. The tool uses a **flag schema** plus per-dialogue usage.

### Playthrough state

In **Play** mode, the tool tracks where you are in the tree and which flags apply for that run-through.

## Design principles

1. **Same data, multiple lenses** — Graph, Yarn, and Play stay in sync because they share one `DialogueTree`.
2. **Fast iteration** — Local persistence and undo/redo support small edits without losing work.
3. **Engine-ready export** — Yarn Spinner output is the primary path to Unreal.
