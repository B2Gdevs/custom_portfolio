# Architecture

## Where the code lives

| Piece | Location |
| --- | --- |
| Shipped app route | `apps/portfolio/app/apps/dialogue-forge/page.tsx` |
| Page state (undo/redo, hydration, export, shortcuts) | `apps/portfolio/lib/dialogue-forge/use-dialogue-forge-page-state.ts` |
| Persisted dialogue + flags | `apps/portfolio/lib/stores/dialogue-forge-store.ts` |
| Editor implementation | `packages/dialogue-forge/` (e.g. `DialogueEditorV2`, React Flow graph, Yarn + Play views) |

The app composes **`DialogueEditorV2`** with props for the active mode (graph-only features such as layout strategy, guide, and flag manager hooks are passed only in graph mode).

## Data model (canonical)

Types live in `packages/dialogue-forge/src/types`. The tree is JSON-serializable and matches what the editor reads and writes.

### Node kinds

Nodes use `type: 'npc' | 'player' | 'conditional'` (see `NODE_TYPE` in the package). NPC nodes hold line content and default routing; player nodes hold **choices**; conditional nodes hold **conditional blocks** for branching logic.

### Sketch (abbreviated)

```typescript
interface DialogueTree {
  id: string;
  title: string;
  startNodeId: string;
  nodes: Record<string, DialogueNode>;
}

interface DialogueNode {
  id: string;
  type: 'npc' | 'player' | 'conditional';
  content: string;
  x: number;
  y: number;
  speaker?: string;
  characterId?: string;
  choices?: Choice[];
  nextNodeId?: string;
  setFlags?: string[];
  conditionalBlocks?: ConditionalBlock[];
  // ...
}

interface Choice {
  id: string;
  text: string;
  nextNodeId?: string;
  conditions?: Condition[];
  setFlags?: string[];
}

interface Condition {
  flag: string;
  operator: /* see CONDITION_OPERATOR in package */;
  value?: boolean | number | string;
}
```

For the full definitions (including `ConditionalBlock` and operators), open the package sources under `packages/dialogue-forge/src/types/`.

## Yarn export and import (implementation)

| Function | File | Role |
| --- | --- | --- |
| `exportToYarn` | `packages/dialogue-forge/src/lib/yarn-converter.ts` | Walks `DialogueTree` nodes and emits Yarn node titles, NPC/player lines, `->` choices, `<<jump>>`, and `<<set $flag>>` style commands from flags and conditions |
| `importFromYarn` | same | Parses `.yarn` text back into a `DialogueTree` (used when the user picks a file in **Yarn** mode) |

**Runtime vs file:** In Yarn Spinner, variable values live in the engineŌĆÖs variable storage; the `.yarn` file carries commands and structure. The converter maps Dialogue Forge flags to `$variable` names in the exported textŌĆösee comments at the top of `yarn-converter.ts`.

**Portfolio wiring:** `useDialogueForgePageState` implements `handleExportYarn` (blob download). `DialogueEditorV2` calls that hook in graph and yarn modes; **Yarn** view also wires **Import .yarn** ŌåÆ `importFromYarn` ŌåÆ `onChange`.

## UI stack

- **Graph** ŌĆö React Flow nodes and edges; layout helpers under `packages/dialogue-forge/src/utils/`.
- **Yarn / Play** ŌĆö Same `DialogueTree`; Yarn view renders exporter-oriented text; Play view simulates flow.

## Persistence

The portfolio store persists **`dialogueTree`** and **`flagSchema`** to `localStorage` under a single key (with migration from older separate keys). Undo/redo for the dialogue tree is managed in **`useDialogueForgePageState`** and is separate from the editorŌĆÖs internal React Flow history.
