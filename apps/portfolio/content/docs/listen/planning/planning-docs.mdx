## Owner workflow

| Step | Rule |
| --- | --- |
| `1` | author or revise fallback rows in `listen-catalog.ts` when shaping new data locally |
| `2` | run `pnpm --filter @portfolio/app listen:seed` to upsert those rows into Payload under the owner tenant |
| `3` | runtime surfaces (`/listen`, home tracks, command-palette search) prefer DB-backed rows first and fall back to file-authored rows only when the DB path is empty or unavailable |
| `4` | private owner-only rows stay `visibility: private`; public preview rows may still use `lockGroup` if they intentionally keep the shared-password flow |
