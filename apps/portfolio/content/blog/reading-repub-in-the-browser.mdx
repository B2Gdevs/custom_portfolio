The [Books](/books) page lets you open a `.repub` file directly in the browser. The viewer fetches the file, unpacks it with JSZip, and runs the bundled React app inside a **sandboxed iframe**. Here's how it works and what to watch for.

## Flow

1. **Fetch**: GET `/books/<bookSlug>/book.repub` (static file from `public/books/`).
2. **Unpack**: JSZip loads the array buffer and exposes entries: `repub.json`, `content/index.html`, `assets/*.js`, `assets/*.css`.
3. **Blob URLs**: Each file is read as a Blob and assigned a blob URL with `URL.createObjectURL(blob)`. We keep a list so we can revoke them on unmount.
4. **Rewrite HTML**: The entry `content/index.html` references `../assets/...`. We replace those paths with the corresponding blob URLs so that when the document loads, script and link tags point at our blobs.
5. **Inject**: We create a Blob for the rewritten HTML and set the iframe's `src` to that blob URL. The iframe loads the document; the document loads scripts and styles from the other blob URLs.
6. **Sandbox**: The iframe has `sandbox="allow-scripts"` only. So the content runs in an opaque origin, can execute JS, but cannot access the parent window or escape the sandbox.

## Security

- **Arbitrary JS**: A `.repub` runs whatever JavaScript was bundled. The reader treats it as untrusted unless you only ever load your own content.
- **Sandbox**: We do **not** add `allow-same-origin`; that would let the content break out. So the iframe cannot access cookies, localStorage, or the parent. Scripts run in isolation.
- **Scope**: First version assumes `.repub` files are served from your own origin (e.g. `public/books/`). Allowing user-uploaded `.repub` would need the same sandboxing plus trust and validation.

## Limitations

- **Blob URL lifetime**: We revoke blob URLs when the component unmounts so we don't leak.
- **No network**: The sandboxed iframe can still make network requests unless you add CSP. For maximum isolation you could serve the iframe from a strict CSP that blocks fetch/XHR; we don't do that by default.
- **Size**: Large bundles (React + Tailwind + deps) mean larger `.repub` files. Tree-shaking and careful dependencies help.
